## 1. Введение

Алгоритм линейной растяжки гистограммы (contrast stretching) – один из базовых методов улучшения качества изображений.
Он преобразует входной диапазон яркостей пикселей [min_val, max_val] в полный диапазон [0, 255].

В работе представлена OpenMP-реализация.
OpenMP позволяет распараллелить как поиск минимума/максимума, так и процесс преобразования пикселей с помощью директив компилятора.
Это даёт возможность получать ускорение на нескольких ядрах,
минимально усложняя код.

## 2. Постановка задачи

Входные данные:
Вектор целых чисел input размера n, где каждый элемент – яркость пикселя (обычно от 0 до 255).

Выходные данные:
Вектор целых чисел output того же размера, содержащий значения после преобразования контрастности.

Преобразование:
Если max_val == min_val – изображение однородное, выход копируется с входа.
Иначе для каждого элемента:

        output[i] = ((input[i] - min_val) * 255) / (max_val - min_val)

Ограничения:

- n > 0

- Все значения лежат в диапазоне, допускающем целочисленное умножение без переполнения.

## 3. Базовый алгоритм (SEQ)

Последовательная версия выполняет два линейных прохода:

1. Поиск min_val и max_val с помощью std::ranges::minmax_element.

2. Преобразование каждого пикселя по формуле растяжки гистограммы.

Детальное описание приведено в seq/report.md.

## 4. Схема распараллеливания

OpenMP-реализация использует два параллельных региона:

4.1 Параллельный поиск минимума и максимума.

    #pragma omp parallel for default(none) shared(input, size) reduction(min : min_val) reduction(max : max_val)
    for (size_t i = 0; i < size; ++i) {
        min_val = std::min(input[i], min_val);
        max_val = std::max(input[i], max_val);
    }

reduction(min:min_val) – каждый поток вычисляет локальный минимум, затем они объединяются.

reduction(max:max_val) – аналогично для максимума.

default(none) – явное указание области видимости переменных.

4.2 Обработка однородного изображения.
При max_val == min_val выполняется параллельное копирование:

    #pragma omp parallel for default(none) shared(input, output, size)
    for (size_t i = 0; i < size; ++i) {
        output[i] = input[i];
    }

4.3 Параллельное преобразование контрастности.

    const int range = max_val - min_val;
    #pragma omp parallel for default(none) shared(input, output, size, min_val, range)
    for (size_t i = 0; i < size; ++i) {
        output[i] = ((input[i] - min_val) * 255) / range;
    }

Вынос инварианта range за пределы цикла для оптимизации.

shared – все переменные доступны всем потокам (только чтение, кроме output).

## 6. Результаты

### 6.1 Корректность

Функциональные тесты подтверждают правильность работы для:

- случайных массивов (50 + i % 101);

- однородных изображений (все пиксели равны);

- граничных размеров (1, 2, …, 1024).

Схема проверки:

- Для каждого тестового размера генерируется входной вектор по формуле 50 + (i % 101).

- Вычисляется эталонный результат по формуле преобразования.

- Результат последовательной реализации сравнивается с эталоном.

Результаты: все тесты пройдены. Расхождений между ожидаемым и фактическим выходом, а также между SEQ и OMP не обнаружено.

## 7. Выводы

OpenMP-версия алгоритма повышения контрастности:

1) полностью корректна - результаты совпадают с корректной последовательной версией;
2) даёт хорошее ускорение для данной задачи.

С увеличением числа потоков сохраняется как минимум двойное ускорение и снижается эффективность, что связано с накладными
расходами на редукцию и барьеры. Таким образом,
OMP даёт выигрыш, когда вычислительные работы превышают накладные расходы.
Полученные результаты дают основу для дальнейшего расчёта ускорения и эффективности параллельных версий.
