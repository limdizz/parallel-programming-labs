## 1. Введение

Алгоритм линейной растяжки гистограммы (contrast stretching) – один из базовых методов улучшения качества изображений.
Он преобразует входной диапазон яркостей пикселей [min_val, max_val] в полный диапазон [0, 255].

В работе представлена последовательная реализация, используемая для последующего сравнения
с параллельными версиями (OpenMP, STL, TBB, MPI+STL). Все замеры ускорения
и эффективности в дальнейшем будут нормированы относительно времени работы этой реализации.

## 2. Постановка задачи

Входные данные:
Вектор целых чисел input размера n, где каждый элемент – яркость пикселя (от 0 до 255).

Выходные данные:
Вектор целых чисел output того же размера, содержащий значения после преобразования контрастности.

Преобразование:
Если max_val == min_val – изображение однородное, выход копируется с входа.
Иначе для каждого элемента:

        output[i] = ((input[i] - min_val) * 255) / (max_val - min_val)

Ограничения:

- n > 0

- Все значения лежат в диапазоне, допускающем целочисленное умножение без переполнения.

Крайние случаи:

- Пустой вектор: валидация возвращает false, задача не выполняется.

- Однородное изображение (все пиксели равны): выходная копия идентична входу.

- Размер n = 1: единственный пиксель преобразуется сам в себя.

## 3. Последовательный алгоритм

Используется простой двухпроходный метод:

1. Поиск минимума и максимума – один линейный проход (std::ranges::minmax_element).

2. Обработка особого случая – если min == max, изображение однородно, выходная копия равна входу.

3. Преобразование каждого пикселя – второй линейный проход с применением формулы.

## 4. Implementation Details

- **Заголовочный файл:** `ops_seq.hpp`.
- **Файл реализации:** `ops_seq.cpp`.

Класс наследуется от ppc::task::Task<InType, OutType> и переопределяет четыре виртуальных метода:

- ValidationImpl - проверяет, что входной вектор не пуст;
- PreProcessingImpl - выделяет память под выходной вектор того же размера;
- RunImpl - основное вычисление (min/max + преобразование);
- PostProcessingImpl - проверяет соответствие размеров выходного и входного векторов.

## 6. Результаты

### 6.1 Корректность

Функциональные тесты подтверждают правильность работы для:

- случайных массивов (50 + i % 101);

- однородных изображений (все пиксели равны);

- граничных размеров (1, 2, …, 1024).

Схема проверки:

- Для каждого тестового размера генерируется входной вектор по формуле 50 + (i % 101).

- Вычисляется эталонный результат по формуле преобразования.

- Результат последовательной реализации сравнивается с эталоном.

Результаты: все тесты пройдены. Расхождений между ожидаемым и фактическим выходом не обнаружено.
