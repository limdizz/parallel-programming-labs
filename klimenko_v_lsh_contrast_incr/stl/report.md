## 1. Введение

Алгоритм линейной растяжки гистограммы (contrast stretching) – один из базовых методов улучшения качества изображений.
Он преобразует входной диапазон яркостей пикселей [min_val, max_val] в полный диапазон [0, 255].

В работе представлена STL-реализация с использованием нативных потоков C++ (std::thread).
Данный подход обеспечивает низкоуровневый контроль над параллельным выполнением без использования внешних библиотек.

## 2. Постановка задачи

Входные данные:
Вектор целых чисел input размера n, где каждый элемент – яркость пикселя (обычно от 0 до 255).

Выходные данные:
Вектор целых чисел output того же размера, содержащий значения после преобразования контрастности.

Преобразование:
Если max_val == min_val – изображение однородное, выход копируется с входа.
Иначе для каждого элемента:

        output[i] = ((input[i] - min_val) * 255) / (max_val - min_val)

Ограничения:

n > 0

Все значения лежат в диапазоне, допускающем целочисленное умножение без переполнения.

## 3. Базовый алгоритм (SEQ)

Последовательная версия выполняет два линейных прохода:

1. Поиск min_val и max_val с помощью std::ranges::minmax_element.

2. Преобразование каждого пикселя по формуле растяжки гистограммы.

Детальное описание приведено в seq/report.md.

## 4. Схема распараллеливания

STL-реализация использует ручное управление потоками через std::thread и статическое распределение данных между потоками.

4.1 Вспомогательная функция GetThreadRange.

        void GetThreadRange(size_t tid, size_t total, size_t num_t, size_t &b, size_t &e) {
            size_t chunk = total / num_t;
            b = tid * chunk;
            e = (tid == num_t - 1) ? total : b + chunk;
        }

Принцип работы:

chunk – базовый размер блока для каждого потока.

Начало блока: b = tid * chunk.

Конец блока: для последнего потока – total, иначе b + chunk.

4.2 Параллельный поиск минимума и максимума.

        std::vector<int> local_min(num_t);
        std::vector<int> local_max(num_t);

        for (int tid = 0; tid < num_t; tid++) {
            threads.emplace_back([&, tid]() {
                size_t begin, end;
                GetThreadRange(tid, total_size, num_t, begin, end);
                
                int current_min = input[begin];
                int current_max = input[begin];
                
                for (size_t i = begin; i < end; i++) {
                    current_min = std::min(current_min, input[i]);
                    current_max = std::max(current_max, input[i]);
                }
                
                local_min[tid] = current_min;
                local_max[tid] = current_max;
            });
        }

        for (auto &th : threads) {
            th.join();
        }

        const int global_min = *std::ranges::min_element(local_min);
        const int global_max = *std::ranges::max_element(local_max);

Принцип работы:

- Каждый поток обрабатывает свой непрерывный блок данных.

- Результаты сохраняются в векторы local_min и local_max.

- После завершения всех потоков вычисляются глобальные экстремумы.

4.3 Обработка однородного изображения.
При global_max == global_min выполняется параллельное копирование:

        if (global_max == global_min) {
            std::ranges::copy(input, output.begin());
            return true;
        }

4.4 Параллельное преобразование контрастности.

        for (int tid = 0; tid < num_t; tid++) {
            threads.emplace_back([&, tid]() {
                size_t begin, end;
                GetThreadRange(tid, total_size, num_t, begin, end);
                
                for (size_t i = begin; i < end; i++) {
                    output[i] = ((input[i] - global_min) * 255) / (global_max - global_min);
                }
            });
        }

        for (auto &th : threads) {
            th.join();
        }

Ключевые особенности STL-подхода:

- Ручное управление потоками – полный контроль над созданием и синхронизацией.

- Статическое распределение – фиксированное разбиение данных на блоки равного размера.

- Отсутствие внешних зависимостей – только стандартная библиотека C++.
