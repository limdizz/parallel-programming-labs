## 1. Введение

Алгоритм линейной растяжки гистограммы (contrast stretching) – один из базовых методов улучшения качества изображений.
Он преобразует входной диапазон яркостей пикселей [min_val, max_val] в полный диапазон [0, 255].

В работе представлена TBB-реализация с использованием библиотеки Intel Threading Building Blocks.
TBB обеспечивает высокоуровневый параллелизм на основе задач с автоматической балансировкой нагрузки.

## 2. Постановка задачи

Входные данные:
Вектор целых чисел input размера n, где каждый элемент – яркость пикселя (обычно от 0 до 255).

Выходные данные:
Вектор целых чисел output того же размера, содержащий значения после преобразования контрастности.

Преобразование:
Если max_val == min_val – изображение однородное, выход копируется с входа.
Иначе для каждого элемента:

        output[i] = ((input[i] - min_val) * 255) / (max_val - min_val)

Ограничения:

n > 0

Все значения лежат в диапазоне, допускающем целочисленное умножение без переполнения.

## 3. Базовый алгоритм (SEQ)

Последовательная версия выполняет два линейных прохода:

1. Поиск min_val и max_val с помощью std::ranges::minmax_element

2. Преобразование каждого пикселя по формуле растяжки гистограммы

Детальное описание приведено в seq/report.md.

## 4. Схема распараллеливания

TBB-реализация использует два основных алгоритмических шаблона: parallel_reduce и parallel_for.

4.1 Параллельный поиск минимума и максимума (parallel_reduce).
        struct MinMax {
            int min_val;
            int max_val;
        };

        MinMax result = tbb::parallel_reduce(
            tbb::blocked_range<size_t>(0, size),
            MinMax{.min_val = input[0], .max_val = input[0]},
            [&](const tbb::blocked_range<size_t> &r, MinMax local) {
                for (size_t i = r.begin(); i < r.end(); ++i) {
                    local.min_val = std::min(local.min_val, input[i]);
                    local.max_val = std::max(local.max_val, input[i]);
                }
                return local;
            },
            [](const MinMax &a, const MinMax &b) {
                return MinMax{
                    .min_val = std::min(a.min_val, b.min_val),
                    .max_val = std::max(a.max_val, b.max_val)
                };
            }
        );

Принцип работы:

- blocked_range – рекурсивное разбиение диапазона на поддиапазоны.

- Тело лямбды – вычисление локальных min/max в каждом поддиапазоне.

- Функция редукции – объединение результатов от разных потоков.

4.2 Обработка однородного изображения (parallel_for).
При max_val == min_val выполняется параллельное копирование:

        tbb::parallel_for(tbb::blocked_range<size_t>(0, size), 
            [&](const tbb::blocked_range<size_t> &r) {
                for (size_t i = r.begin(); i < r.end(); ++i) {
                    output[i] = input[i];
                }
            });

4.3 Параллельное преобразование контрастности (parallel_for).

        tbb::parallel_for(tbb::blocked_range<size_t>(0, size), 
            [&](const tbb::blocked_range<size_t> &r) {
                for (size_t i = r.begin(); i < r.end(); ++i) {
                    output[i] = ((input[i] - min_val) * 255) / (max_val - min_val);
                }
            });

## 7. Результаты

### 7.1 Корректность

Функциональные тесты подтверждают правильность работы для:

- случайных массивов (50 + i % 101);

- однородных изображений (все пиксели равны);

- граничных размеров (1, 2, …, 1024).

Схема проверки:

- Для каждого тестового размера генерируется входной вектор по формуле 50 + (i % 101).

- Вычисляется эталонный результат по формуле преобразования.

- Результат последовательной реализации сравнивается с эталоном.

Результаты: все тесты пройдены. Расхождений между ожидаемым и фактическим выходом, а также между SEQ и OMP не обнаружено.

## 8. Выводы

TBB-версия алгоритма повышения контрастности:

1) полностью корректна - результаты совпадают с корректной последовательной версией;
2) демонстрирует хорошее ускорение для регулярных вычислительных задач.

По результатам тестов на производительность эффективность данной реализации на малом количестве потоков
достаточно высокая, но с увеличением их числа снижается сильнее, чем OpenMP. Ускорение также выходит меньше, чем у OMP,
что делает TBB менее эффективным алгоритм распараллеливания для данной задачи, чем OpenMP.
Полученные результаты дают основу для дальнейшего расчёта ускорения и эффективности параллельных версий.
